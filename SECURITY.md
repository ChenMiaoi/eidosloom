# Security Policy

## Supported Versions

Eidosloom is experimental. Security fixes target the latest tagged release and `main`.

## Reporting A Vulnerability

Please report security issues privately to the maintainer before opening a public issue.

- GitHub: `ChenMiaoi`
- Email: `chenmiao.ku@gmail.com`

Include:

- affected version or commit;
- reproduction steps;
- expected and actual behavior;
- any sensitive data exposure risk.

## Scope

Eidosloom skills may prepare text for ChatGPT web. They must not send secrets, credentials, private customer data, authentication cookies, session stores, or other sensitive data without explicit user authorization.

The browser automation flow depends on the user's logged-in ChatGPT web session and Codex Chrome Extension. Do not bypass login, CAPTCHA, browser permission prompts, or account access controls.
